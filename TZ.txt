Техническое Задание: Project Chimera (v2.4.4 Production Build)
Дата: 30 января 2026 г.
Статус: Утверждено.
Исполнитель: Команда разработки Chimera Solutions (AI Integrated).
1. Цель проекта и Обзор (Team Lead Summary)
Проект «Chimera» — автономный комплекс для конвертации дизайн-макетов в функциональный веб-интерфейс.
Приватность: Локальные вычисления без Cloud API.
Универсальность: Обработка файлов до 400MB (PSD, PDF, FIG, PNG).
Автоматизация: Генерация кликабельных ссылок и карт сайта (sitemap.xml).
2. Стек Технологий (Full-stack Developer Scope)
Компонент	Технология	Обоснование
Ядро	Python 3.12+ (LTS)	Высокая производительность и поддержка библиотек.
Интерфейс	PySide6 (Qt)	Стабильный UI, работа в отдельных потоках (QThread).
OCR / ИИ	PaddleOCR (Local)	Распознавание текста на PNG и восстановление слоев.
Парсинг PDF	pdfminer.six	Векторное извлечение текста и координат объектов.
Парсинг PSD/FIG	psd-tools / zip+json	Глубокий разбор байт-кода и бинарных структур.
Мониторинг	psutil + PowerShell CIM	Аппаратный контроль ресурсов (CPU > 85% — Alert).
3. Архитектура и Отказоустойчивость
ChimeraEngine: Мозг системы. Поддерживает Chunked Loading для работы с файлами >300MB.
MonitorThread: Параллельный сбор метрик (температура, CPU).
Error Handling: Изоляция процессов. Ошибка в одном макете (например, битый PDF) не прерывает общую очередь.
4. Функциональные требования
4.1. Обработка макетов и Граф связей
PDF Векторизация: Прямое извлечение текста и координат (без потери качества). Каждая страница PDF = новая страница сайта.
Semantic Anchor: ИИ-сопоставление PNG-скриншотов с существующим графом проекта.
Slugify: Метод приведения имен слоев (btn_Order -> order.html) к единому стандарту.
4.2. Пользовательский интерфейс (UI Designer Scope)
3 Зоны: Мониторинг ресурсов, Таблица анализа (Граф связей), Лог событий.
CMS Adapter: Выбор из 22 пресетов (WordPress, Bitrix, Tilda, Laravel, React и др.).
Render Logic: Интеграция Jinja2 для генерации адаптивного HTML (Desktop/Mobile).
4.3. Ресурсы и Типографика (Export Service)
100% Persistence: Сохранение всех изображений в ./Export/assets/.
Font Engine: Автоматическое извлечение шрифтов и генерация fonts.css с правилами @font-face.
Style Extraction: Сбор HEX-цветов и CSS-параметров из метаданных макета.
5. Результат поставки и Развертывание
Файлы: Единый main_app.py с интегрированной логикой всех модулей.
Сборка: Standalone-пакет через PyInstaller (--onedir).
Условие: Запуск от имени Администратора для работы CIM-датчиков температуры.
Заключение: Система готова к обработке предоставленного резюме в форматах PSD, FIG, PDF и PNG с автоматической генерацией навигации и сохранением всех ресурсов.
