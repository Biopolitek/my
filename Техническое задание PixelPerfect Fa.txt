ТЗ на создание клона программы (на случай сбоя ветки "Something Went Wrong")
Это ТЗ описывает независимый, параллельный проект, который можно начать с чистого листа, если текущий код станет нестабильным.
"Project Chimera" Техническое Задание
1. Цель проекта
Разработка дублирующей, стабильной автоматизированной системы конвертации дизайн-макетов (PSD, FIG, PDF, PNG) в высокоточный адаптивный веб-интерфейс, ориентированной на отказоустойчивость и модульность.
2. Основной Стек Технологий
Ядро: Python 3.12+ (LTS версия для стабильности).
UI: PyQt/PySide6 (Вместо customtkinter для лучшей кроссплатформенности и богатого функционала).
PSD-Парсинг: psd-tools (с использованием Pillow-SIMD для ускорения рендера).
OCR: Google Cloud Vision API или PaddleOCR (Вместо Tesseract для повышения точности распознавания Rus/Eng).
Шаблонизация: Jinja2 (без изменений).
Сборка: PyInstaller (LTS версии).
3. Модульность и Отказоустойчивость
Модульный дизайн: Система должна быть разделена на микросервисы: UI_Service, Parser_Service, Render_Service, CMS_Mapper. Коммуникация между ними через очередь задач (Celery/Redis) для параллельной обработки.
Обработка ошибок: Вместо общего try-except реализовать логирование ошибок на уровне слоя (Warning), а не краш всего файла.
Docker: Финальная версия должна быть развернута в Docker-контейнере для гарантии идентичности среды на любой машине (решает проблему E:\Python314 vs AppData).
4. Функциональные требования (Расширенные)
Sitemap & Навигация: Автоматический анализ имен слоев (header, footer, nav) для создания работоспособной навигации и генерации sitemap.xml и robots.txt.
Поддержка Шрифтов: В UI добавить выбор Google Fonts или загрузку файла .ttf/.woff, который будет внедрен в HTML-шаблон через @font-face.
Рендеринг PDF/FIG: Использовать pdfminer.six и API Figma (если возможно) для сохранения векторной информации, а не простого растрового изображения.
CMS Mapping GUI: В интерфейсе добавить возможность редактирования префиксов путей для каждой из 20 CMS.
Мониторинг: Добавление модуля мониторинга RAM/CPU/Disk space в реальном времени.
5. Результат поставки
Исходный код на GitLab.
Инструкция по развертыванию через Docker Compose.
Упакованный EXE через PyInstaller (как временное решение).
