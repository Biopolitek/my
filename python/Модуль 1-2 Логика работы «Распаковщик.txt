Модуль 1: Логика работы «Распаковщика» (Экспортер v3.5.1)
Этот модуль выполняет роль «Цифрового криминалиста», извлекая данные из бинарных контейнеров.
Бинарная деконструкция (Magic Bytes): Программа не доверяет расширениям. Она читает первые байты каждого ресурса. Если внутри .fig найден JFIF, файл сохраняется как .jpg для сохранения оригинального цветового профиля (ICC).
Геометрический слепок: Для каждого слоя вычисляется Bounding Box (X, Y, W, H). Эти данные переводятся в CSS-правила с абсолютным позиционированием.
Векторный синтез:
Для PDF: Страница конвертируется в SVG-команды.
Для FIG: Все найденные в бинарном файле canvas пути (paths) объединяются в единый Master SVG Atlas. Это позволяет ИИ-сборщику видеть всю графику в единой системе координат.
Нормализация типографики: Программа извлекает технические имена шрифтов (PostScript Name) и сверяет их с системными. В карту активов заносится статус: INSTALLED или MISSING.
Изоляция страниц: Каждая страница (или фрейм) получает свой index.html и styles.css. Это позволяет ИИ-сборщику работать с сайтом покомпонентно.
Модуль 2: Логика «Сборщика сайта» (Инструкция для ИИ-кодера)
Передайте это описание ИИ (например, GPT-4o или Claude 3.5), чтобы он собрал сайт из ваших файлов.
ТЗ для ИИ-разработчика:
«Тебе предоставлен пакет ресурсов Pixel-Perfect v3.5.1. Твоя цель — собрать фронтенд, который попиксельно совпадает с эталоном из /data/reference/.
Алгоритм твоей работы:
Настройка окружения:
Создай проект (Next.js, Astro или чистый HTML/CSS).
Подключи шрифты из списка map_assets.txt (статус MISSING — повод подключить аналог из Google Fonts).
Добавь глобальный стиль: * { box-sizing: border-box; } и -webkit-font-smoothing: antialiased;.
Послойная сборка (Absolute Layering):
Используй data/index_[page].html как структуру DOM.
Примени стили из data/styles_[page].css.
Важно: Сначала добейся 100% совпадения через position: absolute. Проверь качество, наложив полупрозрачный рендер из /data/reference/ поверх своей верстки.
Семантизация и рефакторинг:
Проанализируй имена классов. Если класс содержит btn, button — замени тег на <button>. Если link, nav — на <a>.
Сгруппируй элементы с общими координатами в Flexbox или Grid контейнеры.
Векторы из assets/vectors/fig_master_vectors.svg переноси инлайново для возможности управления их цветом через CSS.
Интерактивность:
Найди в map_assets.txt раздел [INTERACTIVE ELEMENTS].
Для этих слоев пропиши псевдоклассы :hover и :active.
Выбор CMS:
Если в макете много текстовых блоков — используй WordPress (раздели слои на блоки Gutenberg).
Если это сложный интерактивный лендинг — используй Astro (для максимальной скорости).»
Аналитическое заключение
Данная логика исключает «человеческий фактор». Разработчику не нужно открывать исходный макет, так как программа перевела субъективное «визуальное восприятие» в объективные бинарные данные и числа.
Вопросы перед завершением:
Требуется ли добавить в логику сборщика автоматическое создание адаптивных Media Queries (например, если найдены слои с пометкой MOBILE)?
Нужно ли генерировать JS-скрипт для плавной прокрутки между страницами (если это One-Page сайт)?
Если текущей глубины достаточно, проект готов к финальной упаковке в EXE.
