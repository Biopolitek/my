Техническое Задание: «Project Chimera» (v2.3 Final)
Тип системы: Автономный локальный комплекс конвертации дизайн-макетов.
1. Цель проекта
Создание стабильного программного клона для высокоточной конвертации макетов (PSD, PDF, PNG, FIG) в веб-интерфейс с использованием локального искусственного интеллекта и графового анализа связей.
2. Технологический Стек (Утвержденный)
Ядро: Python 3.12+ (LTS).
Интерфейс: PySide6 (Qt) — реализация через QMainWindow и QThread.
Парсинг:
PSD: psd-tools (локальный разбор байт-кода).
PDF: pdfminer.six (векторное извлечение текста).
Интеллект (OCR): PaddleOCR (локальные веса, работа без API и интернета).
Безопасность: psutil (мониторинг системных метрик).
Сборка: PyInstaller (режим --onedir).
3. Архитектурные особенности
Модульность: Код разделен на независимые классы:
ChimeraEngine: Мозг системы (OCR + Анализ).
MonitorThread: Параллельный поток слежения за ресурсами.
ChimeraApp: Слой визуализации (UI).
Отказоустойчивость: Изоляция процессов через try-except. Ошибка в одном файле (например, битый PDF) логируется в QTextEdit, не прерывая анализ остальных макетов.
4. Функциональные требования (Реализованные)
4.1. Sitemap & Graph Engine (Карта и Связи)
Slug-нормализация: Метод slugify приводит все имена к единому виду, игнорируя префиксы btn_, nav_.
Граф переходов: Автоматическое связывание файлов. Если на странице Home найден текст/слой Contacts, система прокладывает связь к файлу contacts.psd.
Пре-анализ: Вывод таблицы с количеством слоев, семантическими тегами (header, footer) и графом связей до начала рендера.
4.2. Мониторинг ресурсов (System Watcher)
Real-time метрики: Вывод в UI текущей нагрузки CPU, RAM, Disk и Температуры.
Визуальные алерты: Изменение цвета статус-бара на красный при критической нагрузке (>85%), сигнализирующее о необходимости приостановки (Throttling).
4.3. CMS & Assets
CMS Mapping: Выбор из 20 предустановленных префиксов (WordPress, Bitrix, Laravel и др.) для корректной генерации путей к изображениям.
Векторная обработка: Сохранение текстовой структуры PDF вместо растрирования.
5. Инструкция по развертыванию и сборке
5.1. Локальный запуск
Установка зависимостей: pip install PySide6 psutil psd-tools jinja2 pdfminer.six paddleocr.
Запуск: python main_app.py.
5.2. Сборка Standalone (Windows)
Сборка в одну папку для максимальной стабильности:
bash
python -m PyInstaller --noconfirm --onedir --windowed --name "Chimera_Factory_v2.3" --collect-all paddleocr --collect-all pywt --collect-all skimage --collect-all imgaug --hidden-import "PySide6.QtCore" --hidden-import "PySide6.QtWidgets" "main_app.py"
Используйте код с осторожностью.

5.3. Изоляция (Docker)
Финальная стадия: создание контейнера на базе python:3.12-slim с установкой libGL для работы PaddleOCR.
