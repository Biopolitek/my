На основе полученной информации от распаковщика (переданный как параметр путь для распаковки ресурсов, файл шаблона) анализатор выполняет следующие задачи:
глубоко сканирует pdf, fig или psd (последний файл шаблона сайта, с которым работал распаковщик) и соответствующую папку с файлами (результат работы распаковщика ) на предмет наличия ресурсов в шаблоне, которые не удалось передать в папку распаковщику или не в полной мере. Если такое установлено, анализатор делает пометку в текстовом файле ( путь для распаковки ресурсов) , оставляя как замечания от разработчика по ресурсам. Если расхождений нет (все ресурсы обработаны на 100%, то пишется текст в файле - данные с шаблона переданы на 100%) 

Анализатор запускается с правами администратора.
В свою очередь в распаковщике внести изменения после завершения распаковки запустить анализатор (Gui приложение с параметрами(переданный как параметр путь для распаковки ресурсов, файл шаблона), чтобы анализатору было с чем работать Анализатор является последней инстанцией перед передачи в разработку материала, поэтому анализатор ещё и проверяет наличие созданной карты сайта по ресурсам от распаковщика, если её нет, он создаёт её вместе с замечаниями в текстовом файле (пути, переданном от распвковщика). В GUI будет поле для выдачи текущий температуры (результат команды shell  $t = (Get-CimInstance -Namespace root/wmi -ClassName MsAcpi_ThermalZoneTemperature).CurrentTemperature; if($t) { $c = ($t/10 - 273.15); '+' + ('{0:N1}' -f $c) } else { "N/A" }).
