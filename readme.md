# Asset-to-Code Pipeline (ACTE)

### 🎯 Суть проекта
Автоматизированный конвейер для трансформации графических шаблонов в Pixel-Perfect верстку. Система состоит из двух независимых модулей, решающих задачу перехода от сырых ассетов к чистому коду.

### 🏗 Архитектура системы (Модульность)

1.  **Module 1: Resource Unpacker (Распаковщик)**
    *   **Статус:** Beta (Ready 90%+).
    *   **Функционал:** Автоматическая экстракция данных из шаблонов (Figma-embedded, JPG, PNG, SVG, CSS).
    *   **Результат:** Генерация структурированной карты ресурсов и распределение файлов по папкам для последующей сборки.
    
2.  **Module 2: Web Decoder (Декодер)**
    *   **Статус:** In Development.
    *   **Функционал:** На основе карты ресурсов и пользовательских пресетов (настроек) генерирует итоговую структуру сайта.
    *   **Логика:** Учитывает семантику, группировку слоев и типографику.

### 🧠 Моя роль и AI-Driven подход
Проект реализован в рамках концепции **AI-Assisted Engineering**. 
*   **Архитектура:** Я спроектировал логические ветки и связи между модулями.
*   **Управление:** ИИ использовался как инструмент генерации кода по моим жестким спецификациям (ТЗ).
*   **Качество:** Основной акцент сделан на достижении Pixel-Perfect точности при автоматизированной сборке.

### 🛠 Технологический стек
*   **Backend:** Python (Логика парсинга и оркестрация модулей).
*   **Frontend logic:** HTML5/CSS3 (Absolute Positioning & Flexbox mapping).
*   **Graphics:** Обработка растровых и векторных данных (SVG/PNG/WebP).

### 🚀 Как использовать
1. Запустить **Распаковщик** для подготовки ресурсов из шаблона.
2. Сформировать карту активов (`map_assets.txt`).
3. Запустить **Декодер** с выбранными предпочтениями сборки.
